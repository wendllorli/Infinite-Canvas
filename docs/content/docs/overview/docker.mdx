# Docker 部署

如果你希望在自己的机器或服务器上运行项目，可以直接使用 Docker Compose。

根目录 Compose 同时启动画布、Duomi Adapter 和同域 Nginx。先复制环境变量模板并填写真实 `DUOMI_API_KEY`：

```bash
cp duomi-adapter/.env.example .env
```

## 使用发布镜像

```bash
git clone git@github.com:basketikun/infinite-canvas.git
cd infinite-canvas
docker compose up -d
```

启动后访问：

```text
http://localhost:3000
```

## 本地构建镜像

如果需要基于当前源码构建镜像：

```bash
docker compose -f docker-compose.local.yml up -d --build
```

## 文档站镜像

文档站位于 `docs/` 目录，按带服务端能力的 Next.js standalone 应用单独构建，不打进主应用镜像。

使用发布镜像：

```bash
cd docs
docker compose up -d
```

基于当前源码本地构建：

```bash
cd docs
docker compose -f docker-compose.local.yml up -d --build
```

## 数据说明

画布、我的素材和生成记录仍默认保存在浏览器本地。普通渠道的 API Key 仍由浏览器保存；Duomi 真实 Key 只注入 Adapter 容器。生产环境还应通过登录、VPN 或外部网关保护 `/api/duomi/`，避免公开访问者消耗额度。
